# Клонирование OmniRoute репозитория

## Когда это может понадобиться

- Для изучения исходного кода OmniRoute
- Для чтения полной документации
- Если понадобится модифицировать или разрабатывать OmniRoute
- Для просмотра примеров и best practices

## Текущая ситуация

Сейчас проект использует **готовый Docker образ** `diegosouzapw/omniroute:latest`. Это означает, что:
- Сервис полностью рабочий ✓
- Исходный код находится внутри образа
- Для запуска/использования нет необходимости клонировать репо
- Данные хранятся в volume `omniroute_omniroute-data`

## Если потребуется клонировать

### Вариант 1: Отдельная папка (РЕКОМЕНДУЕТСЯ)

```bash
git clone https://github.com/diegosouzapw/omniroute ~/omniroute-source
cd ~/omniroute-source
```

**Преимущества:**
- Независимый от основного проекта
- Простая работа с git
- Легко обновлять отдельно

### Вариант 2: Как подмодуль

```bash
git submodule add https://github.com/diegosouzapw/omniroute docs/omniroute-source
git submodule update --init --recursive
```

Клонируйте только если:
- Нужна документация для понимания, как OmniRoute работает

- Нужны примеры интеграции

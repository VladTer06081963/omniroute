## Prerequisites

Before you begin, make sure you have the following installed on your system:

- **A code editor** (e.g., Visual Studio Code)
- **A terminal** or command prompt
- **Git** (optional, but recommended)

---

## Installation

### Step 1: Download Node.js

Visit the official [Node.js website](https://nodejs.org/) and download the **LTS (Long Term Support)** version for your operating system.

### Step 2: Verify Installation

Open your terminal and run the following commands to confirm that Node.js and npm are installed:

```bash
node -v
npm -v
```

You should see version numbers for both, for example:

```
v20.11.0
10.2.0
```

> **Note:** `npm` (Node Package Manager) is bundled with Node.js and is installed automatically.

## Initializing the Project

### Step 1: Create a Project Directory

```bash
mkdir my-node-project
cd my-node-project
```

### Step 2: Initialize the Project

Run the following command to create a `package.json` file:

```bash
npm init
```

You will be prompted to fill in details such as:

| Field         | Description                        | Default          |
| ------------- | ---------------------------------- | ---------------- |
| `name`        | The name of your project           | directory name   |
| `version`     | Starting version                   | `1.0.0`          |
| `description` | A brief description of the project | (empty)          |
| `entry point` | The main file of your project      | `index.js`       |
| `author`      | Your name                          | (empty)          |
| `license`     | The project license                | `ISC`            |

> **Tip:** To skip the prompts and accept all defaults, use `npm init -y`.

### Step 3: Review `package.json`

After initialization, your `package.json` should look similar to this:

```json
{
  "name": "my-node-project",
  "version": "1.0.0",
  "description": "A sample Node.js project",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "Your Name",
  "license": "ISC"
}
```

## Managing Dependencies

### Installing Dependencies

Dependencies are external packages your project needs to run. Install them using:

```bash
# Install a production dependency
npm install express

# Install multiple dependencies at once
npm install dotenv cors mongoose
```

### Installing Dev Dependencies

Dev dependencies are packages only needed during development (e.g., testing tools, linters):

```bash
npm install --save-dev nodemon eslint jest
```

### Understanding `package.json` After Installing Dependencies

```json
{
  "name": "my-node-project",
  "version": "1.0.0",
  "description": "A sample Node.js project",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "Your Name",
  "license": "ISC",
  "dependencies": {
    "cors": "^2.8.5",
    "dotenv": "^16.3.1",
    "express": "^4.18.2",
    "mongoose": "^8.0.3"
  },
  "devDependencies": {
    "eslint": "^8.56.0",
    "jest": "^29.7.0",
    "nodemon": "^3.0.2"
  }
}
```

### Useful npm Commands

```bash
# Uninstall a package
npm uninstall <package-name>

# Update all packages
npm update

# List installed packages
npm list

# Install all dependencies from package.json (e.g., after cloning a repo)
npm install
```

> **Important:** Always add `node_modules/` to your `.gitignore` file. This folder is auto-generated by `npm install` and should never be committed to version control.

```bash
echo "node_modules/" >> .gitignore
```

## Creating the Server

### Step 1: Create the Entry Point File

Create a file named `index.js` in your project root:

```bash
touch index.js
```

### Step 2: Write the Server Code

Add the following code to `index.js`:

```javascript
const express = require("express");
const cors = require("cors");
require("dotenv").config();

// Initialize the app
const app = express();

// Middleware
app.use(cors());
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// Define a port
const PORT = process.env.PORT || 3000;

// Define a basic route
app.get("/", (req, res) => {
  res.json({ message: "Welcome to My Node.js Server!" });
});

// Define an example API route
app.get("/api/status", (req, res) => {
  res.json({
    status: "running",
    timestamp: new Date().toISOString(),
  });
});

// Start the server
app.listen(PORT, () => {
  console.log(`Server is running on http://localhost:${PORT}`);
});
```

### Step 3: Create a `.env` File (Optional)

Create a `.env` file to store environment variables:

```bash
touch .env
```

Add your configuration variables:

```env
PORT=3000
NODE_ENV=development
```

> **Reminder:** Add `.env` to your `.gitignore` to keep sensitive data out of version control.

## Running the Server

### Option 1: Run Directly with Node

```bash
node index.js
```

You should see:

```
Server is running on http://localhost:3000
```

### Option 2: Run with Nodemon (Recommended for Development)

Nodemon automatically restarts your server when it detects file changes.

Update the `scripts` section in your `package.json`:

```json
"scripts": {
  "start": "node index.js",
  "dev": "nodemon index.js",
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

Then run:

```bash
# For production
npm start

# For development (auto-restarts on changes)
npm run dev
```

### Step 3: Test the Server

Open your browser or use `curl` to verify the server is working:

```bash
# Test the home route
curl http://localhost:3000/

# Test the status route
curl http://localhost:3000/api/status
```

**Expected responses:**

```json
// GET /
{ "message": "Welcome to My Node.js Server!" }

// GET /api/status
{ "status": "running", "timestamp": "2024-01-15T10:30:00.000Z" }
```

## Troubleshooting

### Common Issues and Solutions

1. **`command not found: node`**
   - Node.js is not installed or not added to your system PATH.
   - Reinstall Node.js from the [official website](https://nodejs.org/).

2. **`EADDRINUSE: address already in use :::3000`**
   - Another process is already using port 3000.
   - Change the port in your `.env` file, or kill the existing process:
     ```bash
     # Find the process using port 3000
     lsof -i :3000
     # Kill it
     kill -9 <PID>
     ```

3. **`MODULE_NOT_FOUND`**
   - A required package is missing. Run:
     ```bash
     npm install
     ```
